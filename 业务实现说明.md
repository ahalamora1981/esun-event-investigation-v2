# 业务实现说明

## 系统用途

本系统用于调查交易事件，自动分析事件发生前后的通信记录，识别与事件相关的记录并评估潜在风险。系统帮助业务人员快速定位关键证据，提高事件调查效率。

## 系统工作流程

### 第一步：定义调查事件
在调查开始前，需要明确以下信息：
- **事件名称**：被调查的事件名称（如：230205（23 国开 205））
- **事件时间**：事件发生的准确时间
- **内部人员**：参与该事件的内部员工ID
- **外部人员**：与该事件相关的外部人员姓名

### 第二步：系统自动检索记录
系统自动从以下渠道获取指定时间段内的所有通信记录：
- 通话录音
- 邮件往来
- QTrade聊天记录
- Ideal聊天记录

查询范围包括：
- 参与该事件的内部员工的所有通信记录
- 指定的时间段（事件发生当天或前后几天）

### 第三步：智能评分筛选
系统对每条通信记录进行三维度评分：

#### 时间相关性（30%）
- 记录时间与事件时间重合：100分
- 记录发生在事件之后：每小时扣减20分
- 记录发生在事件之前：每小时扣减40分（双倍惩罚）

#### 用户相关性（30%）
- 内部用户匹配：记录的内部员工ID在事件内部人员列表中，得100分
- 外部用户匹配：记录的外部人员在事件外部人员名单中，得100分
- 最终得分 = （内部得分 + 外部得分）/ 2

#### 内容相关性（40%）
- 使用AI模型分析记录内容与事件名称的关联度
- 得分范围：0-100分

### 第四步：筛选高风险记录
- 只保留综合得分≥50分的记录
- 按时间顺序排列，便于追踪事件脉络

### 第五步：风险评估
对筛选出的每条记录，系统自动分析并输出：
- **风险等级**：高、中、低
- **风险描述**：说明该记录存在什么风险，可能造成什么影响

### 第六步：结果呈现
系统以表格形式展示分析结果，包含每条记录的关键信息，便于快速查看和决策。

## 使用指南

### 准备工作
1. 确认要调查的事件信息
2. 收集参与该事件的内部员工ID和外部人员姓名
3. 确定需要查询的时间范围

### 运行系统
系统由技术人员执行，业务人员提供以下信息：

**事件信息示例：**
```
事件名称：230205（23 国开 205）
事件时间：2026-01-19 14:00
内部用户：1772917751770292225
外部用户：韩梅梅、周子航
```

**查询范围示例：**
```
参与人：1772917751770292225
时间范围：2026-01-19 00:00 至 2026-01-19 23:59
```

### 查看结果
系统运行完成后，会在终端输出表格结果。表格包含以下列：

| 列名 | 说明 |
|-----|------|
| 编号 | 记录序号 |
| 时间 | 记录发生时间 |
| 渠道 | 通信渠道（CALL/EMAIL/QTRADE/IDEAL） |
| 内部用户 | 参与该记录的内部员工姓名 |
| 外部用户 | 参与该记录的外部人员姓名 |
| 时间相关性 | 时间维度评分（0-100） |
| 用户相关性 | 用户维度评分（0-100） |
| 内容相关性 | 内容维度评分（0-100） |
| 总体相关性 | 综合得分（0-100） |
| 风险等级 | 高/中/低 |
| 风险描述 | 风险详情说明 |

### 结果解读

#### 综合得分说明
- **90-100分**：与事件高度相关，需重点关注
- **70-89分**：与事件较相关，建议查看
- **50-69分**：可能与事件相关，作为参考
- **低于50分**：与事件无关，已被系统自动过滤

#### 风险等级说明
- **高**：记录内容可能存在违规、欺诈等严重问题
- **中**：记录内容存在潜在风险，需要进一步确认
- **低**：记录内容风险较低，但仍需关注

#### 决策建议
1. 优先查看高风险等级的记录
2. 重点关注综合得分高的记录
3. 按时间顺序阅读记录，还原事件全貌
4. 结合风险描述，判断是否需要采取进一步措施

## 实际应用场景

### 场景一：交易异常调查
当某笔交易出现异常时，使用本系统调查相关人员在该时间段的通信记录，快速定位可能的问题。

### 场景二：合规审查
定期对关键交易事件进行审查，检查是否存在违规操作或不当沟通。

### 场景三：风险监控
对高风险事件进行持续监控，跟踪相关人员的沟通情况。

## 关键注意事项

### 数据准确性
- 确保提供的事件时间准确，否则会影响评分结果
- 外部人员姓名必须与记录中的姓名完全匹配
- 内部员工ID需正确无误

### 时间范围设置
- 建议查询事件前后1-3天的记录，避免遗漏重要信息
- 时间范围过大会增加处理时间
- 时间范围过小可能遗漏关键记录

### 结果解读
- 系统评分仅供参考，最终判断需结合人工分析
- 风险描述由AI生成，可能存在误差，需人工核实
- 不同渠道的权重相同，但实际重要性可能因业务而异
